# checkin_behavior_analysis_workflow

## Name

checkin_behavior_analysis_workflow

## Module

Module 8: life check-in behavior analysis.

## Backend caller

Spring Boot calls this workflow through `DifyService.callCheckinAnalysis`.

## Inputs

The backend passes these fixed `inputs` fields:

- `user_profile`
- `latest_health_data`
- `risk_result`
- `life_plan`
- `checkin_records`
- `diet_completion_count`
- `exercise_completion_count`
- `total_days`
- `completion_rate`
- `start_date`
- `end_date`
- `user_notes`

`checkin_records` is a JSON string array. Each item contains:

- `date`
- `diet`
- `exercise`
- `note`

## Outputs

The End node returns:

- `analysis_result`: JSON text generated by the LLM.
- `completion_rate`: workflow code-node completion rate.

`analysis_result` should be a JSON object with:

- `completion_rate`
- `diet_summary`
- `exercise_summary`
- `habit_score`
- `life_evaluation`
- `main_problems`
- `improvement_suggestions`
- `next_focus`
- `summary`

The backend persists `completion_rate` using its own calculated value, even if the workflow output differs.
