# 模块5知识库覆盖清单

**建议标签**：覆盖清单, 模块5, Agent能力范围

**主要来源编号**：项目功能文档 + S01-S17

## 模块5需要覆盖的问题类型

模块5的AI医生咨询要支持多轮对话，问题来源不仅是单纯科普，还包括用户健康档案、健康数据、风险评估结果、生活方案、打卡分析和健康资讯追问。因此知识库应按“能被用户自然问到的问题”组织，而不是只按医学教材章节组织。

## 覆盖矩阵

| 模块5能力 | 对应知识库文档 |
|---|---|
| 糖尿病基础知识问答 | 02、03、05 |
| 1型、2型、妊娠糖尿病解释 | 03、13 |
| 风险评估结果解释 | 04、05、16 |
| 血糖、血压、BMI、HbA1c等指标解释 | 04、09、12 |
| 饮食建议 | 06、07、15、16 |
| 运动建议 | 08、12、16 |
| 日常生活习惯建议 | 05、12、16 |
| 个性化生活方案追问 | 06、08、12、16 |
| 打卡分析结果解释 | 12、16、17 |
| 健康资讯内容解释 | 02-15 |
| 高风险/异常情况安全提醒 | 01、09、10、18 |
| 常见误区问答 | 15 |

## 推荐最小导入集

演示时间紧时，至少导入：01、02、03、04、06、08、09、10、11、15、16、18。这样能覆盖基础问答、指标解释、饮食运动、低/高血糖、并发症、误区和安全边界。

## 完整导入集

正式演示建议导入 01-18 全部文件，并把 `source_index.csv` 作为资料来源说明保留在仓库或文档目录中。Dify 内部可以不导入 `source_index.csv`，但答辩时可展示其来源管理。
