# Dify 知识库导入说明

**建议标签**：Dify, 知识库, 导入, 模块5

**主要来源编号**：项目功能文档 + S01-S17

## 资料包用途

本资料包用于搭建“糖尿病预治智能助手”的模块5：AI医生咨询知识库。建议把本目录下的 Markdown 文件作为 Dify Knowledge Base 文档导入，再绑定到 `diabetes_ai_doctor_agent`。

## 导入方式建议

1. 在 Dify 创建知识库，名称建议为：`diabetes_prevention_care_kb` 或 `糖尿病预治知识库`。
2. 文档类型选择普通文本/Markdown。
3. 建议先导入 01-18 号分文件，不建议同时导入 `all_in_one_for_dify.md`，避免重复召回。
4. 切片策略建议：按标题层级或 500-900 中文字符切片，切片重叠 80-150 字。
5. 检索策略建议：优先使用混合检索或向量检索+关键词重排；Top-K 可先设为 4-6。
6. Agent 系统提示词中必须保留：不确诊、不处方、严重异常建议就医、结合用户上下文但不编造未提供信息。

## 文档覆盖范围

- 糖尿病基础知识、类型、糖尿病前期；
- 筛查诊断指标、血糖/A1C/BMI/血压等解释；
- 饮食、碳水、餐盘法、水果和常见食物升糖风险；
- 运动干预、安全边界与运动禁忌；
- 低血糖、高血糖、酮体和就医提醒；
- 慢性并发症预防、肾脏/眼底/足部/心血管随访；
- 体重、血压、血脂、睡眠、心理、戒烟限酒等生活习惯；
- 妊娠、老年、儿童青少年、1型糖尿病等特殊人群；
- 药物与胰岛素的常识性解释边界；
- 常见误区问答、项目上下文回答模板、召回测试样例。

## 使用边界

这些文档是知识库摘要版，不是完整临床指南。真实上线或答辩演示前，建议由有医学背景的老师或医生做一次审核。涉及药物、胰岛素剂量、妊娠、儿童、急症、严重低血糖、高血糖伴酮体等问题时，Agent 应转向“线下就医/联系医生/急诊”而不是继续给出具体治疗方案。
