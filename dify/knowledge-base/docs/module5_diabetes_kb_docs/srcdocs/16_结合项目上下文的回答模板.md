# 结合项目上下文的回答模板

**建议标签**：上下文, 健康档案, 风险评估, 生活方案, 打卡分析

**主要来源编号**：项目功能文档 + S01-S17

## 适用问题

本文件用于让AI医生在系统中结合用户上下文回答，而不是只做泛泛科普。

## 可用上下文类型

后端可传入：

- `user_profile_summary`：年龄、性别、身高、体重、腰围、家族史、慢病史；
- `latest_health_data_summary`：血糖、血压、HbA1c、饮食运动状态；
- `latest_risk_result`：风险等级、主要风险因素、就医提醒；
- `latest_life_plan_summary`：饮食计划、运动计划、每日任务；
- `latest_checkin_summary`：近7天或30天饮食/运动打卡完成情况。

## 回答模板：解释风险评估

“根据你当前系统中的评估结果，你被提示为【风险等级】。这不等于确诊，而是说明你提供的年龄、体重、血糖、血压、家族史或生活习惯中存在一些需要关注的因素。建议你先重点处理【主要风险因素】，并按提示补充或复查相关指标。”

## 回答模板：解释生活方案

“这个生活方案的目标是【plan_goal】。饮食部分重点是稳定主食份量、增加蔬菜和优质蛋白；运动部分建议从安全、可坚持的强度开始。如果你有心脏病、高血压、足部问题或低血糖风险，运动强度需要更保守，并先咨询医生。”

## 回答模板：解释打卡分析

“你最近的打卡完成率是【completion_rate】。饮食方面完成较好/不足的是【diet_summary】，运动方面是【exercise_summary】。接下来建议把目标缩小到一两个具体动作，例如饭后散步10-20分钟、减少含糖饮料、每餐先保证半盘蔬菜。”

## 回答模板：用户数据不足

“目前系统里你的健康档案或最新健康数据还不完整，我可以先给你一般建议，但更准确的建议需要补充年龄、身高体重、腰围、空腹血糖、餐后血糖、血压、家族史和既往病史等信息。”

## 回答模板：严重异常

“你提供的信息里有需要重视的异常情况。AI不能远程判断病情严重程度，建议你尽快联系医生或到线下医疗机构评估。如果伴随胸痛、意识不清、严重低血糖、高血糖伴酮体阳性、持续呕吐或呼吸异常，请及时急诊处理。”

## 不要编造

如果上下文没有给出某项信息，不能写“你有高血压”“你正在用胰岛素”“你最近血糖控制不好”。只能说“如果你有……情况，需要……”。
