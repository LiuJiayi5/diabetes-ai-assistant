# 创新功能联调测试用例

适用于 `innovation` 分支：风险预测 RAG、趋势可视化、管理端相似案例 Top-K。

## 1. 风险预测 + 知识库 RAG

| 编号 | 场景 | 操作 | 预期 |
|------|------|------|------|
| RAG-01 | 正常评估 | 患者 `li_ming` 登录 → 风险预测 → 重新发起评估 | 评估成功，详情页展示风险等级/分数 |
| RAG-02 | 参考依据 | 打开最新评估详情 | 出现「知识库参考依据」列表（2～4 条） |
| RAG-03 | Dify 工作流 | Dify 中确认工作流为：开始 → 知识检索 → LLM → 输出 | 知识检索节点已绑定 `diabetes_knowledge_base` |
| RAG-04 | 失败兜底 | 关闭 Dify 后发起评估 | 返回 502 或友好错误，不出现白屏 |

## 2. 趋势可视化（患者端）

| 编号 | 场景 | 操作 | 预期 |
|------|------|------|------|
| VIZ-01 | 风险趋势 | `li_ming` 打开风险预测页 | 底部「趋势可视化」出现风险分数折线图 |
| VIZ-02 | 指标趋势 | 同一页面继续向下 | 至少展示空腹血糖/体重/腰围中 1 条折线 |
| VIZ-03 | 无数据用户 | 新注册用户（无历史） | 不展示趋势区块或图表为空，不报错 |
| VIZ-04 | 录入后刷新 | 录入新健康数据并重新评估 | 趋势图数据点增加或数值更新 |

## 3. 管理端 Top-K + 趋势

| 编号 | 场景 | 操作 | 预期 |
|------|------|------|------|
| ADM-01 | 相似案例 | `system_admin` → 风险评估列表 → 打开 `li_ming` 的评估详情 | 右侧展示 1～3 条相似案例及相似度百分比 |
| ADM-02 | 匹配理由 | 查看相似案例卡片 | 显示「年龄接近 / 空腹血糖接近」等理由 |
| ADM-03 | 管理端趋势 | 同一详情页右侧 | 展示该用户风险分数与指标趋势图 |
| ADM-04 | 参考依据 | 管理端评估详情正文区 | 同步展示知识库参考依据 |

## 4. 演示推荐路径（答辩 3 分钟）

1. 患者端 `li_ming / 123456`：风险预测 → 发起评估 → 查看详情（参考依据）
2. 返回风险预测页：展示趋势图
3. 管理端 `system_admin / 123456`：打开同一评估 → 展示 Top-K 相似案例 + 趋势图
